# VibeHealth Investor Pitch Deck

An attractive, interactive HTML-based pitch deck designed to showcase VibeHealth to potential investors.

## Features

✨ **Professional Design**
- Premium "Soft Pop" aesthetic with warm gradients and organic shapes
- Fully responsive and mobile-optimized
- Smooth transitions and animations
- Carefully crafted color schemes per slide

🎯 **9 Compelling Slides**
1. **Hero** - Eye-catching introduction with bunny mascot
2. **Problem** - Four key pain points in the health app market
3. **Solution** - VibeHealth's unique value proposition
4. **Features** - Six core feature pillars
5. **Design** - Premium design philosophy and branding
6. **Roadmap** - Five-phase product development plan
7. **Tech Stack** - Modern, scalable technology choices
8. **Market Opportunity** - Traction metrics and vision
9. **Call to Action** - Contact information and closing pitch

🎮 **Interactive Navigation**
- Click arrow buttons to navigate between slides
- Use keyboard arrows (← →) for quick navigation
- Current slide indicator shows progress
- Disabled navigation at start/end

📱 **Responsive Layout**
- Works on desktop, tablet, and mobile
- Optimized grid layouts that adapt to screen size
- Touch-friendly controls

## How to Use

### View Locally
```bash
# Open in your default browser
open index.html

# Or run a local server
python -m http.server 8000
# Then visit http://localhost:8000/pitch-deck
```

### Deploy
This is a static HTML file, so it can be deployed to:
- GitHub Pages
- Netlify
- Vercel
- Any static file hosting service

## Customization

Edit `index.html` to customize:
- **Contact information** in Slide 9 (email, website, GitHub)
- **Company metrics** in Slide 8 (numbers and labels)
- **Colors & gradients** in the CSS `<style>` section
- **Copy text** in any slide's HTML

## Browser Support

- Chrome/Edge 90+
- Firefox 88+
- Safari 14+
- Mobile browsers (iOS Safari, Chrome Mobile)

## Tips for Presenting

1. **Full Screen**: Press F11 for full-screen mode
2. **Pacing**: Spend 1-2 minutes per slide
3. **Interaction**: Use animations and hover effects to engage
4. **Backup**: Have PDF version ready in case of technical issues
5. **Practice**: Review before presentation to nail transitions
